2、事件 对象

事件 ： 对某个元素的某种操作 就是一个事件
事件对象 ： 当触发某个事件时，产生的对象 就是事件对象  event使用前提 必须有 事件   不同事件产生的对象不同
事件对象的兼容 ：var e =  evt || event       【注】：向上兼容，先兼容高版本
document.onclick = function(){
        alert(event)//当触发某个事件时，产生的对象 就是事件对象 [objectMouseEvent]
    //但event火狐不识别
    }
document.onclick = function(evt){
    alert(evt)//[object MouseEvent],火狐识别这种
}


3、鼠标事件对象的属性

坐标属性 ： 
pageX / pageY    相对于文档顶部和左侧的坐标    【注】:获取的属性值都是number类型，可以直接使用
clientX / clientY   相对于 document窗口的顶部 和左侧的坐标  如果有滚动条出现  pageY = clientY + 页面滚走的距离scrolltop  
offsetX / offsetY  （用于拖拽效果）  相对于操作元素的内部偏移量
【注】：offsetX/offsetY 火狐不识别  识别layerX/layerY   多以考虑兼容性   layer：层
【注】：兼容性--------offsetX || layerX
console.log(e.pageX,e.pageY)//690 375 据页面顶端的距离
console.log(e.clientX,e.clientY)//690 176 据屏幕顶端的距离
console.log(e.offsetX,e.offsetY)//79 75 据父元素边界的距离
document.onclick = function(e){
    var e = e || event;    //向上兼容  先兼容高版本    //ie  只兼容  event
    console.log("page:" + e.pageX + "," + e.pageY)
    //page是距离页面顶端的距离
    console.log("client:" + e.clientX + "," + e.clientY)
    //client是距离document顶端（左上角）的距离
    console.log( "offset:" + e.offsetX + "," + e.offsetY );
    //offset是距离最近的父元素边界的距离,用于拖拽
}

6、事件流 

当某个事件执行时，从子元素向父元素触发 或 从父元素向子元素触发  称为事件流
事件流的两种模式 ：
事件冒泡 ： 从子元素向父元素触发    【注】:ie的最上层是document谷歌火狐的最上层是window，即相同事件ie冒泡只能延续到document级别
事件捕获 ： 从父元素向子元素触发
事件冒泡 ： 当某个事件触发时  同样的事件会向父元素触发，这个过程就称为 事件冒泡。 
                  不是所有的事件都会产生冒泡问题              onfocus  onblur onload 不会产生冒泡问题 
阻止事件冒泡兼容 ：     propagation:传播        cancel:取消          bubble:气泡
e.stopPropagation ? e.stopPropagation() : e.cancelBubble = true;    【注】:函数名等于函数本身（js解析器知识 ），布尔环境下，有函数为真，为空为假            propagation：传播        cancle：取消
e.stopPropagation();//高版本的阻止冒泡，stoppropagation是事件对象属性
e.cancelBubble = true;//ie的阻止冒泡

9、事件监听   addEventListener()   高版本浏览器  

事件监听好处   1 --- 可以为同样元素绑定多次同一个事件    2---程序员可以使用事件监听的方式 确定 触发的事件过程是冒泡还是捕获
事件源.addEventListener("去掉on的事件",function(){} , true/false)   默认是 false  冒泡

一个元素既有捕获 又有冒泡 先捕获true 后冒泡false        【注】：顺序是父元素捕获---子元素捕获---子元素监听---父元素监听
    【注】：只有在父元素子元素都有同种事件（例如都有点击事件）的情况下会按照上述顺序，如果只有父元素有点击事件，那么将按照书写顺序先执行先写的事件，不论他是监听的方式，还是捕获的方式
                                                            
    //兼容 事件监听函数  ：   扩展 
    // 当一个函数作为参数传递时  这样的参数称为 回调函数 
    function addEvent(obj,type,callback){    
       if( obj.addEventListener ){//高版本 
           obj.addEventListener( type , callback );
       }else{//低版本
           obj.attachEvent( "on" + type , callback );
       }
    }

【注】：当一个参数（实参，形参）是函数的时候，就是回调函数（回头再调用的函数）
