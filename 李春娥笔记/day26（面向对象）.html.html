5、工厂模式 创建对象
//创建工厂
    //工厂 ：  备料---加工---出厂
    function createDog( name , color ){
        //备料 --- 创建一个对象
        var obj = new Object();
        //加工  --- 为对象添加属性或方法
        obj.name = name;
        obj.color = color;
        //出厂 ---返回对象
        return obj;
    }
工厂模式优点 ： 解决了多个同类对象创建时 代码重复的问题 
工厂模式的缺点 ： 
    一般对象的创建是通过new关键字创建的，但是工厂模式  是通过 函数调用方式创建的对象，对象的创建方式不规范
    不能确定某个对象属于哪一个类              
    


通过instanceof可以判断对象属于哪一类   格式   obj instanceof 构造函数，当是属于该构造函数类时，返回true，不是返回false    instance：实例

    var xh = bird("hh",10);
    console.log(xh)//{name: "hh", fly: 10}
    console.log(xh instanceof Object)//true
    console.log(xh instanceof bird)//false


6 、 构造函数 创建对象

为了和普通函数区分 ， 构造函数的命名规范 是 大驼峰的命名规范  QianFengJiaoYu       【注】：通过构造函数new出来的对象叫做实例
         【注】：函数只是函数，当函数经过new后，创建的对象就是属于函数这类，函数也变成了类，原来函数中的this指向函数的调用者，而变成构造函数变成类后，函数中的this指向通过构造函数创建的对象

构造函数中的this  指向的是构造函数 new 出来的对象。
构造函数中的属性  叫做实例属性
构造函数中的方法  叫做实例方法
构造函数的优点 ： 
    对象是通过new关键字创建的，规范性好
    能够确定某个对象属于哪一个类

构造函数的缺点 ： 
    多个同类对象创建时，方法也会被重建，方法不共享    【注】：但是通过构造函数的共同方法prototype可以解决，通过prototype创建的属性和方法都是共享的

    function Stu(name,age){
       this.name = name;
       this.age = age;
       this.state = function(){
           console.log( study);
       }
    }
    var xh = new Stu("xh",18);
    var hg = new Stu("hg",28);
    console.log(xh.state)
    /*
     * ƒ (){
           console.log( study);
       }
     */
    console.log(hg.state)
    /*
     * ƒ (){
           console.log( study);
       }
     */
    console.log( xh.state == hg.state )//false
    //虽然代码块完全一样，但是通过构造函数定义的对象虽然具有一模一样的方法，但是方法私有，分别储存在自己的函数内，即通过构造函数创建的类的方法不共享
    console.log(xh instanceof Stu)//true   通过构造函数创建的对象都属于该类，用instanceof可以判断
    console.log(hg instanceof Stu)//true


6、通过构造函数的原型 创建对象
写在构造函数外面的属性叫做原型属性 ，方法叫做原型方法        prototype：原型，标准      proto：原型
优点： 
    解决了多个同类对象创建时，方法不共享问题（方法共享）    
缺点 ：
    所有的属性名相同  

【注】：所有的构造函数都有原型属性 即   构造函数  .  prototype . 变量名 = 属性/方法，通过这样定义原型属性或者方法

    function Stu(name,age){
       this.name = name;
       this.age = age;
       this.state = function(){
           console.log( study);
       }
    }
    var xh = new Stu("xh",18);
    var hg = new Stu("hg",28);
    
    Stu.prototype.sport = function(){
       return this.name + " like ballgame";
    }
    console.log(xh.sport())//xh like ballgame
    console.log(hg.sport())//hg like ballgame
    console.log(xh.sport == hg.sport)//true
    
    通过原型prototype创建的方法在同一空间存储着，所以相等，因为调用的就是一个方法，通过原型解决了通过构造函数创建的方法不共享的缺点

8、面向对象中的this
构造函数中的this指向构造函数new出来的对象        
原型方法中的this指向构造函数new出来的对象
在定时器中的this指向window对象
在事件中的this指向事件的触发者   
如果在定时器 或 事件中，调用构造函数new出来的对象的属性或方法，必须要改变this指向

【注】：this的指向只有两种
    1、事件的触发者
    2、函数的调用者     PS：定时器是window的方法，所以定时器中的this指向window

bind() 改变匿名函数的this指向         【注】：构造函数中的函数都是匿名函数，因为都是形如  变量 = function(){}  的形式  变量中存放的是该匿名函数的地址
